# Design: Behavior Points System (ClassDojo-style)

**Date:** 2026-03-15
**Status:** Approved
**Part of:** ClassDojo-inspired features (Phase A of 7)

## Context

GrewMe is a kids learning app with radar charts for academic skill tracking. The user wants ClassDojo-inspired features. This spec covers Phase A: the behavior points system — the core ClassDojo experience where teachers award/deduct points for student behaviors in real-time.

This system is **separate** from the existing radar chart / daily score system. Behavior points track classroom conduct; radar scores track academic skills.

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Relationship to radar scores | Separate system | Behavior and academics are different dimensions; mixing them muddies both |
| Category configuration | School-configurable | School manager defines categories; all teachers use the same set for consistency |
| Point values | Weighted (+1 to +5, -1 to -5) | Different behaviors have different significance; school manager sets weights |
| Awarding UI | Both Grid + Quick Bar modes | Grid for individual focus, Quick Bar for batch awarding — teacher picks per situation |
| Projection display | Equal grid with progress bars | Inclusive — shows all students equally without competitive ranking |
| Parent visibility | Weekly summary report | No real-time notifications; parents get a weekly digest |

## Data Model

### BehaviorCategory

School-defined behavior types. Created by school manager.

| Field | Type | Notes |
|---|---|---|
| `id` | bigint | PK |
| `school_id` | bigint | FK → School. Categories are per-school |
| `name` | string | e.g., "Helping Others", "Off-task" |
| `description` | text | Optional longer description |
| `point_value` | integer | Range: -5 to +5. Positive = good behavior, negative = needs improvement |
| `is_positive` | boolean | Derived from point_value > 0, but stored for query convenience |
| `icon` | string | Emoji (e.g., "🤝", "⏰") |
| `color` | string | Hex color for UI (e.g., "#059669") |
| `position` | integer | Sort order in the UI (acts_as_list) |
| `deleted_at` | datetime | Soft delete — preserves historical point records |
| `timestamps` | | created_at, updated_at |

**Validations:**
- `name` unique within school (scoped: `school_id`)
- `point_value` between -5 and +5, not zero
- `icon` present
- `color` present, valid hex

**Associations:**
- `belongs_to :school`
- `has_many :behavior_points`

### BehaviorPoint

Individual point award record. Created when teacher taps to award.

| Field | Type | Notes |
|---|---|---|
| `id` | bigint | PK |
| `student_id` | bigint | FK → Student |
| `teacher_id` | bigint | FK → Teacher (who awarded) |
| `classroom_id` | bigint | FK → Classroom (context of award) |
| `behavior_category_id` | bigint | FK → BehaviorCategory |
| `point_value` | integer | Snapshot of category's point_value at time of award |
| `note` | text | Optional teacher note (e.g., "Helped Budi with math") |
| `awarded_at` | datetime | When the point was awarded (defaults to now) |
| `revoked_at` | datetime | Null unless revoked; set when teacher undoes within 15-min window |
| `timestamps` | | created_at, updated_at |

**Validations:**
- All FK references present
- `point_value` between -5 and +5, not zero
- `awarded_at` present

**Scopes:**
- `active` — where `revoked_at` is null
- `today` — where `awarded_at` is today (school timezone)
- `for_week(date)` — points within the week containing `date`
- `positive` / `negative` — by point_value sign

**Associations:**
- `belongs_to :student`
- `belongs_to :teacher`
- `belongs_to :classroom`
- `belongs_to :behavior_category`

### BehaviorSummary

Cached weekly aggregation. Generated by Solid Queue job.

| Field | Type | Notes |
|---|---|---|
| `id` | bigint | PK |
| `student_id` | bigint | FK → Student |
| `classroom_id` | bigint | FK → Classroom |
| `week_start` | date | Monday of the summary week |
| `total_points` | integer | Net points (positive - negative) |
| `positive_count` | integer | Number of positive point events |
| `negative_count` | integer | Number of negative point events |
| `top_behavior_category_id` | bigint | FK → BehaviorCategory (most frequent positive) |
| `timestamps` | | created_at, updated_at |

**Validations:**
- Unique constraint on `[student_id, classroom_id, week_start]`
- `top_behavior_category_id` is nullable (student may have zero positive points in a week)

**Associations:**
- `belongs_to :student`
- `belongs_to :classroom`
- `belongs_to :top_behavior_category, class_name: 'BehaviorCategory', optional: true`

## GraphQL API

### Mutations

#### `createBehaviorCategory`
- **Auth:** School Manager of the school
- **Input:** `schoolId`, `name`, `description`, `pointValue`, `icon`, `color`
- **Returns:** BehaviorCategoryType
- **Notes:** Auto-sets `is_positive` and appends to end of `position` list

#### `updateBehaviorCategory`
- **Auth:** School Manager of the school
- **Input:** `id`, optional `name`, `description`, `pointValue`, `icon`, `color`
- **Returns:** BehaviorCategoryType

#### `deleteBehaviorCategory`
- **Auth:** School Manager of the school
- **Input:** `id`
- **Returns:** Success boolean
- **Notes:** Soft delete — sets `deleted_at`, preserves historical BehaviorPoints

#### `reorderBehaviorCategories`
- **Auth:** School Manager of the school
- **Input:** `categoryIds` (array of IDs in desired order)
- **Returns:** Array of BehaviorCategoryType
- **Notes:** Updates `position` field for all categories

#### `awardBehaviorPoint`
- **Auth:** Teacher assigned to the classroom
- **Input:** `studentId`, `classroomId`, `behaviorCategoryId`, `note` (optional)
- **Returns:** BehaviorPointType + student's updated daily total
- **Side effects:** Broadcasts to `BehaviorChannel` for projection display
- **Rate limit:** Max 1 point per student per behavior per 30 seconds (prevents double-tap). Enforced via `Rails.cache` with a 30s TTL key: `behavior_point:#{student_id}:#{category_id}`

#### `batchAwardBehaviorPoints`
- **Auth:** Teacher assigned to the classroom
- **Input:** `studentIds` (array), `classroomId`, `behaviorCategoryId`, `note` (optional)
- **Returns:** Array of BehaviorPointType + updated totals
- **Side effects:** Single broadcast to `BehaviorChannel` with all updates
- **Notes:** Used by Quick Bar mode

#### `revokeBehaviorPoint`
- **Auth:** Teacher who awarded the point
- **Input:** `behaviorPointId`
- **Returns:** BehaviorPointType (with `revoked_at` set)
- **Constraints:** Only within 15 minutes of `awarded_at`; after that, returns error
- **Side effects:** Broadcasts revocation to `BehaviorChannel`

### Queries

#### `behaviorCategories(schoolId)`
- **Auth:** Any authenticated user in the school
- **Returns:** Array of BehaviorCategoryType, ordered by `position`
- **Filters:** Excludes soft-deleted categories

#### `classroomBehaviorToday(classroomId)`
- **Auth:** Teacher assigned to the classroom
- **Returns:** Array of `{ student, totalPoints, positiveCount, negativeCount, recentPoints }` for today
- **Notes:** Powers both the awarding page and projection display

#### `studentBehaviorHistory(studentId, startDate, endDate)`
- **Auth:** Teacher of student's classroom OR parent of the student
- **Returns:** Array of BehaviorPointType within date range, ordered by `awarded_at DESC`

#### `classroomBehaviorSummary(classroomId, weekStart)`
- **Auth:** Teacher or School Manager
- **Returns:** Array of BehaviorSummaryType for the classroom for a given week

#### `studentWeeklyReport(studentId)`
- **Auth:** Parent of the student
- **Returns:** BehaviorSummaryType for the current/latest week + comparison to previous week (trend)

#### `classroomBehaviorExport(classroomId, weekStart)`
- **Auth:** School Manager
- **Returns:** CSV-formatted string with columns: student_name, total_points, positive_count, negative_count, top_behavior
- **Notes:** Client downloads via Blob URL

### Subscription

#### `behaviorPointAwarded(classroomId)`
- **Auth:** Teacher assigned to the classroom
- **Payload:** `{ studentId, studentName, behaviorName, pointValue, newDailyTotal, isRevocation }`
- **Channel:** `BehaviorChannel` (ActionCable)
- **Notes:** Broadcasts on award, batch award, and revocation

## Frontend Pages

### Teacher Pages

#### Classroom Behavior Page (`/teacher/classrooms/[id]/behavior`)
Main awarding interface with two tab modes:

**Grid Mode (default):**
- Grid of student cards showing avatar, name, and today's point total
- Tap a student → popup with behavior category buttons (positive on top, negative below)
- Tap a behavior → point awarded instantly, card flashes green/red, total updates
- Undo button appears for 15 seconds after awarding

**Quick Bar Mode:**
- Top bar shows all behavior categories as colored chips
- Tap a behavior chip to select it (stays selected)
- Below: student list with checkboxes
- Tap students to select multiple, then "Award All" button
- Confirmation toast: "Helping Others +3 awarded to 5 students"

**Shared elements:**
- Tab switcher: Grid | Quick Bar
- Date display (today)
- "Open Projection" button → opens display page in new window
- Today's class summary at top: total positive, total negative, net

#### Projection Display (`/teacher/classrooms/[id]/behavior/display`)
- Full-screen, dark gradient background, no navigation
- Equal grid of student cards: avatar, name, daily point total, progress bar
- Real-time updates via `behaviorPointAwarded` subscription
- Animation: when a point is awarded, the student's card briefly glows green (positive) or red (negative), and the number counts up/down
- Press Escape or click to exit full-screen
- Auto-reconnect indicator if WebSocket disconnects

#### Student Behavior Detail (`/teacher/students/[id]/behavior`)
- Tab on existing student detail page
- Behavior history table: date, behavior, points, note, revoke button (if within 15 min)
- Weekly trend chart (bar chart: positive green, negative red, per day)
- Total points this week vs last week

#### Behavior Settings (`/teacher/behavior/settings`)
- Read-only view of school's behavior categories (teachers can't edit, but can see them)
- Links to school manager settings if they have SM role

### School Manager Pages

#### Behavior Categories (`/school/behavior/categories`)
- List of all behavior categories with drag-to-reorder
- Each row: icon, name, point value, color swatch, edit/delete buttons
- "Add Category" button → inline form or modal
- Positive and negative categories shown in separate sections
- Delete shows confirmation: "This will hide the category. Existing points are preserved."

#### School Behavior Dashboard (`/school/behavior`)
- Overview cards: total points awarded this week across all classrooms, most common positive/negative behavior
- Classroom comparison table: classroom name, net points, positive %, teacher
- Sortable: by most improved, needs attention, top performers
- Export as CSV button (uses a `classroomBehaviorExport` query that returns CSV-formatted string; downloaded client-side via Blob)

### Parent Pages

#### Child Behavior Report (`/parent/children/[id]/behavior`)
- Tab on existing child detail page
- Weekly summary card: net points, trend arrow (↑↓→ vs last week), positive count, negative count
- Bar chart: daily point totals for the week (green bars for positive days, red for net-negative)
- Top behaviors list: most frequent positive behaviors with counts
- Teacher notes section (if any notes were attached to points)

### Integration with Existing Pages

- **Teacher Dashboard:** Add a "Behavior" card per classroom showing today's quick stats (net points, # awards given)
- **Parent Dashboard:** Add a behavior widget per child showing this week's net points and trend
- **Student detail pages:** Add "Behavior" tab alongside existing tabs

## Real-time Implementation

### BehaviorChannel (ActionCable)

```ruby
# app/channels/behavior_channel.rb
class BehaviorChannel < ApplicationCable::Channel
  def subscribed
    classroom = Classroom.find(params[:classroom_id])
    # Verify teacher is assigned to this classroom
    if current_user.classrooms.include?(classroom)
      stream_for classroom
    else
      reject
    end
  end
end
```

**Broadcast payload:**
```json
{
  "type": "point_awarded",
  "student_id": 42,
  "student_name": "Andi",
  "behavior_name": "Helping Others",
  "point_value": 3,
  "new_daily_total": 15,
  "is_revocation": false
}
```

### Frontend Subscription (SvelteKit)

The projection display page subscribes using the existing ActionCable client in `front-end/src/lib/actioncable.ts`. On receiving a broadcast, it updates the reactive store for the affected student's card.

## Weekly Report Generation

### Solid Queue Job: `GenerateBehaviorSummariesJob`

- **Schedule:** Runs hourly via Solid Queue recurring job. On each run, processes schools whose local timezone is currently Sunday 20:00 (handles multi-timezone deployments)
- **Logic:**
  1. For each school, for each classroom, for each student enrolled this week:
  2. Aggregate active (non-revoked) BehaviorPoints for the week
  3. Upsert BehaviorSummary record
  4. Create notification for each parent using the existing `Notification` model: "Weekly behavior report for [child] is ready"
- **Idempotent:** Uses upsert on unique constraint `[student_id, classroom_id, week_start]`

## Authorization (Pundit)

### BehaviorCategoryPolicy
- `create?` / `update?` / `destroy?` / `reorder?` — school manager of the school
- `index?` — any authenticated user in the school

### BehaviorPointPolicy
- `create?` — teacher assigned to the classroom
- `revoke?` — the teacher who awarded the point, within 15 minutes
- `index?` (for student) — teacher of student's classroom OR parent of the student

### BehaviorSummaryPolicy
- `show?` (student report) — teacher of student's classroom OR parent of the student
- `index?` (classroom summary) — teacher or school manager
- `export?` — school manager

## Error Handling

| Scenario | Behavior |
|---|---|
| Double-tap (same student+behavior within 30s) | Return error: "Point already awarded recently. Wait before awarding again." |
| Revoke after 15 min | Return error: "Revoke window expired. Points older than 15 minutes cannot be undone." |
| Category deleted | Soft delete — existing points preserved with snapshot `point_value`. Category hidden from UI. |
| No categories configured | Teacher sees setup prompt: "Your school hasn't set up behavior categories yet. Ask your school manager." |
| WebSocket disconnect (projection) | Auto-reconnect with visual "Reconnecting..." indicator. On reconnect, refetch full state via `classroomBehaviorToday` query. |
| Teacher not assigned to classroom | 403 Forbidden via Pundit policy |
| Parent viewing non-child | 403 Forbidden via Pundit policy |

## Testing Strategy

| Layer | Tests |
|---|---|
| **Model** | BehaviorCategory validations (name uniqueness per school, point_value range, icon/color presence). BehaviorPoint associations, scopes (active, today, for_week, positive/negative). BehaviorSummary unique constraint. |
| **Mutations** | Authorization (teacher can only award in own classrooms, SM can only manage own school). Input validation. Batch awarding creates correct number of records. Revoke within/outside window. Soft delete behavior. |
| **Queries** | Scoping (parents only see own children, teachers only see own classrooms). Aggregation correctness in classroomBehaviorToday. Weekly report calculations. |
| **ActionCable** | BehaviorChannel subscription auth (reject unauthorized). Broadcast fires on award/batch/revoke. Payload format correct. |
| **Integration** | Full flow: SM creates categories → teacher awards points → parent views weekly report. |

## Out of Scope (for this spec)

- Student avatars/monsters (Phase C)
- Behavior reports beyond weekly summary (Phase B will add monthly/term reports)
- Push notifications to parents on each point award
- Sound effects on projection display
- Student self-reporting behaviors
- AI-generated behavior insights
