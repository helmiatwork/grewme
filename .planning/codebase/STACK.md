## Platform Requirements

**Development:**
- Ruby 4.0.1 (use rbenv/asdf with `backend/.ruby-version`)
- PostgreSQL (any recent version, 9.3+ per config comments)
- Android Studio or IntelliJ IDEA with KMP plugin
- Xcode (for iOS targets: iosX64, iosArm64, iosSimulatorArm64)
- JDK 17+
- libvips (for image processing)

**Production:**
- Docker (multi-stage Dockerfile at `backend/Dockerfile`)
- PostgreSQL with 4 databases: primary, cache, queue, cable (per `backend/config/database.yml` production section)
- No Redis required - uses Solid Queue/Cache/Cable (all PostgreSQL-backed)
- jemalloc enabled in Docker for memory optimization
- Thruster reverse proxy in front of Puma (exposed on port 80)

**Mobile Targets:**
- Android: minSdk 26 (Android 8.0+), targetSdk 35
- iOS: x64 (simulator), Arm64 (device), Simulator Arm64 (Apple Silicon simulator)
- Teacher app package: `com.grewme.teacher`
- Parent app package: `com.grewme.parent`

## CI/CD

**GitHub Actions** (defined in `backend/.github/workflows/ci.yml`):
- Triggers: push to `main`, all pull requests
- 3 parallel jobs:
  1. `scan_ruby` - Brakeman security scan + Bundler Audit
  2. `lint` - RuboCop with caching
  3. `test` - Rails tests with PostgreSQL service container
- Uses `actions/checkout@v6`, `ruby/setup-ruby@v1`

**Dependabot** (defined in `backend/.github/dependabot.yml`):
- Weekly updates for Bundler dependencies
- Weekly updates for GitHub Actions
- Max 10 open PRs each

**Deployment:**
- Kamal 2.10.1 via `backend/config/deploy.yml`
- Target: single server at `192.168.0.1` (placeholder)
- Docker registry: `localhost:5555` (placeholder)
- amd64 architecture builds
- Solid Queue runs in-process with Puma (`SOLID_QUEUE_IN_PUMA: true`)
- Persistent volume: `grewme_storage:/rails/storage`
