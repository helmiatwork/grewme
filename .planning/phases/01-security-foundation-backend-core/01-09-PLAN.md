2. **Write JwtService tests** (`backend/test/services/jwt_service_test.rb`):
   - Test encode returns string starting with "ey" (base64 JWT format)
   - Test decode returns hash with :user_id matching input
   - Test decode includes :jti (UUID format) and :iat (integer timestamp)
   - Test expired token: encode with `exp: 1.second.ago`, then decode returns nil
   - Test tampered token: modify a character in the token string, decode returns nil
   - Test nil token: decode(nil) returns nil
   - Test empty string: decode("") returns nil

   - Use `ActionDispatch::IntegrationTest` (integration tests hit routes)
   - **Login tests:**
     - Valid login returns 200, body has access_token, refresh_token, expires_in, user
     - Invalid email returns 401 with error JSON
     - Wrong password returns 401
   - **Register tests:**
     - Valid registration creates user, returns 201, body has tokens
     - `assert_difference "User.count", 1` for successful registration
     - Submitting `role: "admin"` still creates a teacher (role ignored)
     - Password < 8 chars returns 422
     - Duplicate email returns 422
   - **Refresh tests:**
     - Login first to get refresh_token, then POST /auth/refresh with it
     - Returns new access_token + refresh_token
     - Old refresh_token is revoked — using it again returns 401
     - Missing refresh_token returns 400
   - **Unauthenticated access:**
     - GET /api/v1/classrooms without Authorization header returns 401
