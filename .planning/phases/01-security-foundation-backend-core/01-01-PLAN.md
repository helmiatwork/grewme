From backend/app/services/jwt_service.rb:
```ruby
class JwtService
  SECRET_KEY = Rails.application.credentials.secret_key_base || ENV.fetch("SECRET_KEY_BASE", "dev-secret-key")
  ALGORITHM = "HS256"

  def self.encode(payload, exp: 24.hours.from_now)
    payload[:exp] = exp.to_i
    JWT.encode(payload, SECRET_KEY, ALGORITHM)
  end

  def self.decode(token)
    decoded = JWT.decode(token, SECRET_KEY, true, algorithm: ALGORITHM)
    HashWithIndifferentAccess.new(decoded.first)
  rescue JWT::DecodeError, JWT::ExpiredSignature => e
    Rails.logger.warn("JWT decode error: #{e.message}")
    nil
  end
end
```

<task type="auto">
  <name>Task 1: Fix JWT secret and harden JwtService</name>
  <files>backend/app/services/jwt_service.rb</files>
  <action>
Rewrite `JwtService` to:

1. **Remove the hardcoded fallback** `"dev-secret-key"`. Replace with:
   ```ruby
   def self.secret_key
     Rails.application.credentials.secret_key_base ||
       raise("SECRET_KEY_BASE not configured. Run `rails credentials:edit`")
   end
   ```
   Use a class method (not a constant) so it doesn't fail at boot during asset precompilation but DOES fail on first use if missing.

2. **Change access token expiry** from 24 hours to 15 minutes (`15.minutes.from_now`) — short-lived tokens, refresh tokens come in Plan 05.

3. **Add JTI (JWT ID)** to payload: `payload[:jti] = SecureRandom.uuid` — needed for future token revocation.

4. **Add IAT (issued at)** to payload: `payload[:iat] = Time.current.to_i`.

5. **Remove role from JWT payload** — the auth controller currently includes `role:` in the JWT. The Authenticatable concern already fetches the user from DB, so role in the token is unnecessary and can become stale. The JwtService itself doesn't add role, but document that callers should only pass `user_id:`.

6. **Keep the same rescue pattern** but use `secret_key` method instead of `SECRET_KEY` constant.
  </action>
  <verify>
    <automated>cd backend && bin/rails runner "JwtService.encode(user_id: 1)" 2>&1 | grep -v "SECRET_KEY_BASE not configured" && echo "PASS" || echo "Expected: raises without credentials OR encodes with credentials"</automated>
  </verify>
  <done>JwtService has no fallback secret, uses 15-min expiry, includes jti/iat in payload, and raises if credentials are missing.</done>
</task>

1. **Fix role escalation in AuthController:**
   - Remove `:role` from `register_params`. Change to: `params.permit(:name, :email, :password, :password_confirmation)`
   - Registration defaults to `teacher` role (already the enum default `teacher: 0`).
   - Remove `role:` from the `JwtService.encode` call — only pass `user_id:`.
   - In the `login` action, also remove `role:` from `JwtService.encode` — only pass `user_id: user.id`.

2. **Wire ApplicationController auth chain:**
   - Add `include Authenticatable` to `ApplicationController`.
   - Add `before_action :authenticate_user!` to `ApplicationController`.
   - The `AuthController` already has `skip_before_action :authenticate_user!, only: [:login, :register]` — this will now work correctly.

3. **Add password complexity to User model:**
   - Add `validates :password, length: { minimum: 8 }, if: -> { new_record? || password.present? }` to User model.
   - The `if` condition ensures password validation only fires on create or when password is being changed.

4. **Improve Authenticatable concern:**
   - Ensure `authenticate_user!` properly handles nil token (when no Authorization header present).
   - The current implementation already handles this via `JwtService.decode(nil)` returning nil, which is fine.
